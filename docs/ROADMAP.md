# 🛸 Watchtower Project Roadmap

이 문서는 **Watchtower** 애플리케이션의 전체 기획 및 구현 로드맵을 정리한 문서입니다. 단순한 상태 체크 도구를 넘어, 강력한 개발자용 테스트 도구로 진화하는 것을 목표로 합니다.

## 🏗️ 1단계: 웹사이트 스테이터스 체크 (진행 중)
웹 인프라의 가용성과 성능을 실시간으로 감시합니다.

- [x] **FE/BE: 기본 HTTP HEAD 요청**
    - 사이트 응답 여부 및 Latency 측정 (완료)
    - JSON 파일 기반 영구 저장 및 임포트/익스포트 (완료)
    - 도메인 병렬 체크를 통한 성능 최적화 (완료)
- [ ] **FE: 상세 점검 확장**
    - 하위 페이지들에 대한 재귀적/선택적 요청 및 상태 점검
    - 최대 콘텐츠 렌더링 시간(LCP) 측정 및 성능 지표 시각화
- [ ] **BE: API 단위 테스트**
    - 특정 엔드포인트에 대한 시나리오 기반 API 유효성 테스트

## 🌐 2단계: 외부 네트워크 프록시 및 로컬 연결
모바일 기기 등 외부 디바이스에서 로컬 개발 환경에 안전하게 접근합니다.

- [ ] **디바이스 연결 브릿지**
    - iOS/Android 앱의 Webview 시작 시 `localhost`로 연결할 수 있는 기능
- [ ] **로컬 SSL 인증서 제공**
    - 로컬 환경에서도 HTTPS 개발이 가능하도록 신뢰할 수 있는 CA 및 인증서 자동 생성/설정

## 🛡️ 3단계: 도메인 프록시 및 트래픽 변조
트래픽을 가로채고 수정하여 다양한 시나리오를 시뮬레이션합니다.

- [x] **도메인 로컬 라우팅 (FE/BE)**
    - 특정 도메인을 로컬 서버로 연결하여 멀티 도메인 기능 테스트 (완료)
    - 엔드포인트 더미 데이터(Mocking) 연결 및 실제 백엔드 전환 테스트
- [ ] **API 로깅·Replay·Mock·문서**
    - 프록시 Request/Response 로깅
    - 로깅된 Request로 테스트 Replay
    - 로깅된 Response로 목데이터 프록시
    - API Schema(OpenAPI) 기반 문서 자동생성
    - 문서 내 API Request 실행 Form
    - Request 양식 기반 테스트 케이스 등록·재실행·회귀 테스트
- [ ] **도메인–프록시 연동 및 검색**
    - 모니터링 도메인 ↔ 프록시 라우트 연동
    - 프록시 라우트 추가 시 도메인 검색/자동완성
    - 통합 검색 (도메인·그룹·프록시 여부)
- [ ] **로그 및 데이터 수집**
    - 이벤트 로그 데이터 수집 및 분석
    - 사용자가 방문하는 웹사이트의 데이터 자동 수집 기능
- [ ] **인증 및 네트워크 제어**
    - 로그인 세션 변조 또는 인증 토큰 조작 기능
    - 네트워크 속도 조절(Throttling)을 통한 저대역폭 환경 시뮬레이션

## 🎭 4단계: Playwright 통합 및 자동화
강력한 E2E(End-to-End) 테스트 엔진을 탑재합니다.

- [ ] **E2E 테스트 인프라**
    - 앱 내에서 Playwright 스크립트 실행 및 결과 리포트
    - 기능별 상세 로깅 및 히스토리 관리
- [ ] **레코딩 및 재현 (Record & Replay)**
    - 사용자 활동을 레코딩하고 이를 재현하는 기능
    - 재현 시 특정 이벤트 후 연결되는 API 호출 및 응답 기록
- [ ] **비주얼 테스트**
    - 화면 스냅샷 및 비교(Diffing) 기능
- [ ] **멀티 디바이스 환경**
    - 다양한 디바이스 해상도 및 브라우저 엔진 환경에서의 테스트 자동화
