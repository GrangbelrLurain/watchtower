# 🗺️ Watchtower Project Roadmap

이 문서는 **Watchtower** 애플리케이션의 전체 기획 및 구현 로드맵을 정리한 문서입니다. 단순한 상태 체크 도구를 넘어, 강력한 개발자용 테스트 도구로 진화하는 것을 목표로 합니다.

## 🟢 1단계: 웹사이트 스테이터스 체크 (진행 중)
웹 인프라의 가용성과 성능을 실시간으로 감시합니다.

- [x] **FE/BE: 기본 HTTP HEAD 요청**
    - 사이트 응답 여부 및 Latency 측정 (완료)
    - JSON 파일 기반 영구 저장 및 익스포트/임포트 (완료)
    - 도메인 병렬 체크를 통한 성능 최적화 (완료)
- [ ] **FE: 상세 지표 확장**
    - 최대 콘텐츠 페인트 시간(LCP) 측정 및 성능 지표 시각화
- [ ] **BE: API 단위 테스트**
    - 특정 엔드포인트에 대한 단일 API 유효성 테스트

## 🟡 2단계: 내부 네트워크 프록시 및 로컬 연결
모바일 기기 및 외부 디바이스에서 로컬 개발 환경에 안전하게 접근합니다.

- [ ] **디바이스 연결 브릿지**
    - iOS/Android 등의 Webview 시작 시 localhost로 연결할 수 있는 기능
- [ ] **로컬 SSL 인증서 제공**
    - 로컬 환경에서도 HTTPS 개발이 가능하도록 도와주는 CA 및 인증서 자동 생성/설정

## 🟠 3단계: 트래픽 프록시 및 테스트 시나리오 파이프라인 (핵심)
단순 모니터링을 넘어, 화면(문서)과 네트워크 계층을 연결하여 자동화된 시나리오 검증 및 모킹 파이프라인을 구축합니다. 아래의 9단계 워크플로우를 통해 구현됩니다.

- [x] **도메인 로컬 라우팅 (FE/BE)**
    - 특정 도메인을 로컬 서버로 연결하여 멀티 도메인 기능 테스트 (완료)
- [ ] **1. 개발 문서 작성 및 관리**
    - OpenAPI 파서를 활용한 API 문서 자동 생성 및 사용자가 직접 수정/추가할 수 있는 살아있는 문서(Living Docs) 환경 구축
- [ ] **2. 각 도메인 별 하위 페이지 리스트업**
    - 루트 도메인 하위에 존재하는 구체적인 라우팅 페이지(예: /login, /cart)들을 트리 구조로 DB화 및 관리
- [ ] **3. 문서에 하위 페이지 링크 추가**
    - 생성된 개발 문서와 실제 프론트엔드 하위 페이지를 서로 링크하여, 특정 기능이 어느 페이지에서 동작하는지 매핑
- [ ] **4. 하위 페이지 HEAD 요청으로 테스트**
    - 리스트업된 각 하위 페이지에 대해 주기적으로 HEAD 요청을 날려 개별 라우트의 생존 여부 및 응답 속도 체크
- [ ] **5. 하위 페이지 별 사용하는 API 리스트업**
    - 로컬 프록시 캡처 기능을 활용하여, 사용자가 특정 하위 페이지에 접속했을 때 발생하는 종속 API 호출 흐름을 자동으로 수집하여 리스트업
- [ ] **6. 하위 페이지 별 테스트 시나리오 구성**
    - 페이지별로 수집된 API 리스트를 바탕으로, 실행 순서(Step 1, Step 2)와 기대 결과(Assertion)를 정의하여 묶어내는 테스트 시나리오 편집기 제공
- [ ] **7. 테스트 시나리오 별 사용하는 API 리스트업**
    - 단일 시나리오 내에서 호출되는 여러 API들의 헤더, 페이로드(Body), 그리고 API 간의 데이터 전달(예: 이전 API의 응답값을 다음 API의 변수로 사용) 관계 정의
- [ ] **8. 시나리오 별 API 결과 확인 (자동 검증)**
    - 구성된 시나리오를 백그라운드에서 순차 실행(Runner)하고, 그 결과를 문서나 대시보드에서 성공/실패 여부와 함께 상세 확인
- [ ] **9. 레코딩 된 결과로 테스트 시나리오 돌릴 때 모킹 및 검증**
    - 정상적으로 성공한 시나리오의 응답 데이터를 영구 저장(Golden Master 레코딩). 이후 프론트엔드 개발 시, 백엔드를 거치지 않고 저장된 응답을 그대로 반환(Mocking)하여 안정적인 UI 테스트 환경 제공 및 스펙 변경 감지

## 🔴 4단계: Playwright 통합 및 자동화
강력한 E2E(End-to-End) 테스트 엔진을 탑재합니다.

- [ ] **E2E 테스트 인프라**
    - 앱 내에서 Playwright 스크립트 실행 및 결과 리포트
    - 기능별 상세 로깅 및 히스토리 관리
- [ ] **이벤트 로그 수집 및 조작**
    - 사용자 네트워크 행동 레코딩 (Record & Replay)
    - 로그/세션 변조, 헤더 조작, 네트워크 속도 조절(Throttling)을 통한 극한 환경 테스트
- [ ] **시각적 회귀 테스트**
    - 화면 스냅샷 및 비교(Diffing) 기능
- [ ] **멀티 디바이스 환경**
    - 다양한 디바이스 해상도 및 브라우저 엔진 환경에서의 테스트 자동화
